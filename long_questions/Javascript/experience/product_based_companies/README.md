## 🎯 Interview Focus Areas (Product Companies)

- ✅ **JS Internals**: Closures, prototype chain, event loop, garbage collection
- ✅ **Async patterns**: Promises, generators, async/await, AbortController, concurrency limits
- ✅ **Performance**: V8 hidden classes, memory leaks, virtual scrolling, rAF, Web Workers
- ✅ **System Design**: Real-time WebSockets, SSR/CSR/ISR trade-offs, micro-frontends
- ✅ **React Deep Dive**: Fiber architecture, hooks internals, Concurrent Mode, React 18
- ✅ **TypeScript**: Generics, conditional types, branded types, discriminated unions
- ✅ **Security**: XSS prevention, CSRF, JWT storage, CSP
- ✅ **Testing**: Jest mocking, React Testing Library patterns, async testing
- ✅ **DSA**: Two Sum, Sliding Window, Tree BFS/DFS, Binary Search, DP, LRU Cache
- ✅ **Machine Coding**: Build production-quality components under time pressure

## 💡 Tips for Product Company JavaScript Interviews

1. **Know the event loop cold** — be able to predict output of complex async code
2. **Implement from scratch** — `bind`, `Promise.all`, `debounce`, `curry` are common asks
3. **V8 optimization** — hidden classes, why to avoid deleting properties
4. **React reconciliation** — understand Fiber, why keys matter, how re-renders propagate
5. **TypeScript depth** — generics, utility types, discriminated unions are frequently asked
6. **Security mindset** — always know where to store JWT tokens and why (XSS vs CSRF tradeoff)
7. **System design** — be ready to design real-time features, caching, state management
8. **Testing best practices** — test behavior not implementation (RTL philosophy)
9. **DSA fundamentals** — practice Two Sum, Sliding Window, BFS/DFS, DP patterns in JS
10. **Machine coding** — practice building full components like Autocomplete and Kanban in 60 min
