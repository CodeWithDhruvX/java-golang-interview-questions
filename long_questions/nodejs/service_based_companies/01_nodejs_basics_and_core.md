## ❓ Frequently Asked Questions

### Q1. What is Node.js and how does it work under the hood?

**Answer:**
Node.js is not a framework or a language; it is an open-source, cross-platform JavaScript runtime environment built on Google Chrome's V8 JavaScript engine.

**Key Characteristics:**
1. **Asynchronous & Event-Driven:** All APIs of the Node.js library are asynchronous (non-blocking). Node.js servers never wait for an API to return data.
2. **Single-Threaded but Highly Scalable:** Node.js uses a single thread with event looping. This mechanism helps the server respond in a non-blocking way, making it highly scalable.
3. **V8 Engine:** It uses the V8 engine to convert JavaScript code into machine code, executing it very fast.
4. **libuv:** A C library that provides the event loop and asynchronous I/O capabilities.

### Q2. Explain the difference between `require` (CommonJS) and `import` (ES Modules)?

**Answer:**

```javascript
// --- CommonJS (Node.js Default historically) ---
// Synchronous loading
const fs = require('fs');

module.exports = function myFunc() {
    console.log("Hello from CommonJS");
};

// --- ES Modules (Modern Standard) ---
// Asynchronous loading, allows tree-shaking
import fs from 'fs';

export function myFunc() {
    console.log("Hello from ES Modules");
}
```

**Key Differences:**
- **Loading:** `require` is synchronous and loads the module at runtime. `import` is asynchronous and loads at parse time.
- **Scope:** `require` uses `module.exports`, while `import` uses `export` and `import` keywords.
- **Top-level await:** Only ES Modules support top-level await.
- **Configuration:** To use ES Modules in Node.js, you must add `"type": "module"` in `package.json` or use the `.mjs` extension.

### Q3. What is the use of `package.json` and `package-lock.json`?

**Answer:**

- **`package.json`:** It is the heart of a Node.js project. It holds metadata about the project (name, version, author), scripts (start, test), and lists the dependencies (and devDependencies) required by the project. The versions here often use semantic versioning with carets (`^`) or tildes (`~`).
- **`package-lock.json`:** This file is automatically generated for any operations where npm modifies either the `node_modules` tree or `package.json`. It describes the exact tree that was generated, such that subsequent installs are able to generate identical trees, regardless of intermediate dependency updates. It locks the versions of the installed packages.

### Q4. Differentiate between `Dependencies` and `DevDependencies`?

**Answer:**

```json
{
  "dependencies": {
    "express": "^4.18.2",
    "mongoose": "^7.0.0"
  },
  "devDependencies": {
    "jest": "^29.5.0",
    "nodemon": "^2.0.22"
  }
}
```

- **Dependencies:** Packages required for the application to run in production (e.g., Express, React, Mongoose). Installed using `npm install <package_name>`.
- **DevDependencies:** Packages needed only for local development and testing (e.g., Nodemon, Jest, ESLint, Webpack). They are not installed when running `npm install --production`. Installed using `npm install <package_name> --save-dev`.

### Q5. Explain the Core Modules in Node.js?

**Answer:**
Node.js comes with several built-in modules that provide essential functionalities without the need to install external packages.

1. **`fs` (File System):** Used to handle file operations like reading, writing, updating, and deleting files.
   ```javascript
   const fs = require('fs');
   const data = fs.readFileSync('/file.md', 'utf8'); // Sync
   fs.readFile('/file.md', 'utf8', (err, data) => { /* Async */ });
   ```

2. **`path`:** Provides utilities for working with file and directory paths.
   ```javascript
   const path = require('path');
   const fullPath = path.join(__dirname, 'folder', 'file.txt');
   ```

3. **`http`:** Used to create HTTP servers and handle network requests.
   ```javascript
   const http = require('http');
   const server = http.createServer((req, res) => {
       res.end('Hello World');
   });
   server.listen(3000);
   ```

4. **`os`:** Provides information about the operating system.
   ```javascript
   const os = require('os');
   console.log(os.cpus().length); // Number of CPU cores
   console.log(os.totalmem());    // Total memory
   ```

### Q6. What is the difference between `__dirname` and `__filename`?

**Answer:**
Both are global variables in Node.js (specifically in CommonJS modules).

- **`__dirname`:** Returns the absolute path of the directory containing the currently executing file.
- **`__filename`:** Returns the absolute path of the currently executing file itself, including the file name.

```javascript
// If file is located at /Users/dev/project/app.js
console.log(__dirname);  // Output: /Users/dev/project
console.log(__filename); // Output: /Users/dev/project/app.js
```
*(Note: These are not available globally in ES Modules. You have to use `import.meta.url` to derive them).*

### Q7. What is an EventEmitter in Node.js?

**Answer:**
Node.js core API is built around an asynchronous event-driven architecture. The `events` module provides the `EventEmitter` class, which allows you to create, fire, and listen for custom events.

```javascript
const EventEmitter = require('events');
const myEmitter = new EventEmitter();

// 1. Register a listener
myEmitter.on('userJoined', (name) => {
    console.log(`Welcome, ${name}!`);
});

// 2. Emit the event
myEmitter.emit('userJoined', 'Alice');
```
Many core modules inherit from `EventEmitter`. For example, a readable stream emits 'data' and 'end' events; an HTTP server emits 'request' events.
