> Companies like **Google, Meta, Atlassian, Airbnb, Netflix, Stripe, Twitter, LinkedIn, Flipkart, Swiggy, Zomato, Razorpay**, etc.

Product-based companies ask **deep, scenario-based questions** on React internals, performance optimization, state management patterns, and frontend system design. Expect **whiteboard-style** architecture discussions, algorithmic thinking applied to UI, and debugging scenarios.

## 🎯 Interview Focus Areas (Product Companies)

- ✅ React Fiber: reconciliation, commit phase, priority scheduling
- ✅ Performance: memoization, code splitting, lazy loading, virtualization
- ✅ State management: Redux Toolkit, Zustand, derived state, selectors
- ✅ Concurrent Mode: Suspense, transitions, deferred values
- ✅ Design patterns: HOC, render props, compound components
- ✅ Testing: unit, integration, E2E — RTL + Cypress
- ✅ Frontend system design: micro-frontends, design systems, CDN strategies
- ✅ Advanced: Server Components, TypeScript integration, Web Vitals

---

## 💡 Tips for Product Company Interviews

1. **Understand React Fiber deeply** — reconciliation, priority, work loop
2. **Master performance patterns** — when to memo, bundle splitting, rendering bottlenecks
3. **Know your state management** — trade-offs between Redux, Zustand, Context
4. **Practice frontend system design** — component libraries, CDN, A/B testing
5. **Test like a pro** — RTL queries, mocking, async testing
6. **TypeScript fluency** — generics with React, utility types
7. **Know Web Vitals** — LCP, CLS, FID and how React impacts them
