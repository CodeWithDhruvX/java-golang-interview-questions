### Q1. What is Jest and how do you run tests?

```bash
# CRA comes with Jest pre-configured
npm test              # runs in watch mode
npm test -- --coverage # see test coverage report
npm test -- --watchAll=false  # run once (CI)

# Vite projects use Vitest (Jest-compatible)
npm install -D vitest
# vitest.config.ts already generated by Vite template
```

```javascript
// Example test file: src/utils/math.test.js
import { add, multiply } from './math';

// describe: group related tests
describe('Math utilities', () => {
  // it / test: individual test case
  it('adds two numbers', () => {
    expect(add(2, 3)).toBe(5);      // exact equality
  });

  it('multiplies numbers', () => {
    expect(multiply(3, 4)).toBe(12);
  });

  it('handles negative numbers', () => {
    expect(add(-1, 5)).toBe(4);
  });
});

// Common Jest matchers:
expect(value).toBe(5)              // ===
expect(value).toEqual({ a: 1 })    // deep equality
expect(value).toBeTruthy()
expect(value).toBeFalsy()
expect(value).toBeNull()
expect(arr).toContain('item')
expect(fn).toHaveBeenCalled()
expect(fn).toHaveBeenCalledWith(arg1)
expect(fn).toThrow('error message')
```

### Q2. How do you write a basic component test with RTL?

```jsx
// src/components/Greeting.jsx
function Greeting({ name }) {
  return <h1>Hello, {name}!</h1>;
}

// src/components/Greeting.test.jsx
import { render, screen } from '@testing-library/react';
import Greeting from './Greeting';

describe('Greeting component', () => {
  it('renders the greeting with name', () => {
    render(<Greeting name="Alice" />);

    // screen.getByText — throws if not found (best for expected elements)
    const heading = screen.getByText('Hello, Alice!');
    expect(heading).toBeInTheDocument();  // from @testing-library/jest-dom
  });

  it('renders correctly with different names', () => {
    render(<Greeting name="Bob" />);
    expect(screen.getByText('Hello, Bob!')).toBeInTheDocument();
  });

  it('renders heading element', () => {
    render(<Greeting name="Alice" />);
    // getByRole is preferred — tests accessibility
    const heading = screen.getByRole('heading', { level: 1 });
    expect(heading).toHaveTextContent('Hello, Alice!');
  });
});
```

### Q3. How do you test user interactions?

```jsx
import { render, screen } from '@testing-library/react';
import userEvent from '@testing-library/user-event';
import Counter from './Counter';

describe('Counter component', () => {
  it('increments count when button clicked', async () => {
    const user = userEvent.setup();
    render(<Counter />);

    const button = screen.getByRole('button', { name: 'Increment' });
    expect(screen.getByText('Count: 0')).toBeInTheDocument();

    await user.click(button);
    expect(screen.getByText('Count: 1')).toBeInTheDocument();

    await user.click(button);
    expect(screen.getByText('Count: 2')).toBeInTheDocument();
  });

  it('resets count when reset button clicked', async () => {
    const user = userEvent.setup();
    render(<Counter />);

    await user.click(screen.getByRole('button', { name: 'Increment' }));
    await user.click(screen.getByRole('button', { name: 'Increment' }));
    expect(screen.getByText('Count: 2')).toBeInTheDocument();

    await user.click(screen.getByRole('button', { name: 'Reset' }));
    expect(screen.getByText('Count: 0')).toBeInTheDocument();
  });

  it('types in text input', async () => {
    const user = userEvent.setup();
    render(<SearchBar />);

    const input = screen.getByRole('textbox', { name: /search/i });
    await user.type(input, 'React');

    expect(input).toHaveValue('React');
  });
});
```

### Q4. What is snapshot testing?

```jsx
// Snapshot testing: capture component output as a text snapshot
// Fails if output changes — catch unexpected regressions

import { render } from '@testing-library/react';
import Button from './Button';

it('matches snapshot', () => {
  const { container } = render(
    <Button variant="primary" size="large">Submit</Button>
  );
  expect(container).toMatchSnapshot();
});

// First run: creates __snapshots__/Button.test.jsx.snap
// Subsequent runs: compares against saved snapshot
// Snapshot file (auto-generated):
// exports[`matches snapshot 1`] = `
// <div>
//   <button class="btn btn-primary btn-large">Submit</button>
// </div>
// `;

// Update snapshots when intentional change occurs:
// npm test -- -u  (or press 'u' in watch mode)

// ⚠️ Snapshot testing limitations:
// - Large snapshots are hard to review
// - Easy to update without reviewing — "just update the snapshot"
// - Better suited for small, stable utility components
// - NOT a substitute for interaction tests
```

### Q5. How do you mock functions in Jest?

```jsx
// jest.fn() — creates a mock function
const onSubmit = jest.fn();

render(<Form onSubmit={onSubmit} />);
await userEvent.click(screen.getByRole('button', { name: 'Submit' }));

// Assert it was called
expect(onSubmit).toHaveBeenCalled();
expect(onSubmit).toHaveBeenCalledTimes(1);
expect(onSubmit).toHaveBeenCalledWith({ name: 'Alice', email: 'alice@test.com' });

// jest.fn() with implementation
const mockFn = jest.fn().mockReturnValue(42);
const mockAsync = jest.fn().mockResolvedValue({ user: 'Alice' });
const mockError = jest.fn().mockRejectedValue(new Error('Failed'));

// Spy on imported function
import * as utils from './utils';
jest.spyOn(utils, 'formatDate').mockReturnValue('Jan 1, 2024');

// Restore original after test
afterEach(() => jest.restoreAllMocks());

// Clear call history between tests
afterEach(() => jest.clearAllMocks());
```

### Q6. What should you test vs. what should you skip?

```jsx
// ✅ DO TEST:
// 1. Component renders correctly with different props
// 2. User interactions produce expected output
// 3. Error states and edge cases
// 4. Important business logic

// Example: test the behavior, not implementation
it('shows error when password is too short', async () => {
  const user = userEvent.setup();
  render(<RegisterForm />);

  await user.type(screen.getByLabelText('Password'), '12');
  await user.click(screen.getByRole('button', { name: 'Register' }));

  expect(screen.getByText('Password must be at least 8 characters')).toBeInTheDocument();
});

// ❌ DON'T TEST:
// 1. Implementation details (which functions are called internally)
// 2. Third-party library behavior (React, axios, etc.)
// 3. Styling/CSS (use visual regression tools like Chromatic)
// 4. Simple pass-through props on dumb components

// Example: testing too many implementation details (bad)
it('calls setState correctly', () => {
  const setState = jest.fn();
  // DON'T test internals like setState calls
  // Test the visible outcome instead
});
```

### Q7. How do you test a form component?

```jsx
import { render, screen, waitFor } from '@testing-library/react';
import userEvent from '@testing-library/user-event';
import LoginForm from './LoginForm';

describe('LoginForm', () => {
  const user = userEvent.setup();
  const mockOnLogin = jest.fn();

  beforeEach(() => {
    render(<LoginForm onLogin={mockOnLogin} />);
  });

  afterEach(() => jest.clearAllMocks());

  it('renders all form fields', () => {
    expect(screen.getByLabelText('Email')).toBeInTheDocument();
    expect(screen.getByLabelText('Password')).toBeInTheDocument();
    expect(screen.getByRole('button', { name: 'Login' })).toBeInTheDocument();
  });

  it('submits with valid credentials', async () => {
    await user.type(screen.getByLabelText('Email'), 'test@example.com');
    await user.type(screen.getByLabelText('Password'), 'password123');
    await user.click(screen.getByRole('button', { name: 'Login' }));

    await waitFor(() => {
      expect(mockOnLogin).toHaveBeenCalledWith({
        email: 'test@example.com',
        password: 'password123'
      });
    });
  });

  it('shows validation error for empty fields', async () => {
    await user.click(screen.getByRole('button', { name: 'Login' }));

    expect(await screen.findByText('Email is required')).toBeInTheDocument();
    expect(mockOnLogin).not.toHaveBeenCalled();
  });
});
```
