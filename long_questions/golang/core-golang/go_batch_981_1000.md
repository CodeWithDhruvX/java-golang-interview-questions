### Question 981: How do you create internal packages in Go?

**Answer:**
Folder named `internal/`.
Go compiler forbids importing `myproject/internal/foo` from `otherproject`.
Enforces boundaries.

### Explanation
Internal packages in Go use the internal/ folder name. The Go compiler forbids importing packages from myproject/internal/foo by other projects, enforcing architectural boundaries and preventing external dependencies on internal implementation details.

### How to Explain in Interview (Spoken style format)
**Interviewer:** How do you create internal packages in Go?
**Your Response:** "I create internal packages by using an `internal/` folder in my project. The Go compiler enforces that only code within the same project can import packages from the internal directory. This means other projects can't import `myproject/internal/foo`, which protects my internal implementation details. I use this for code that I don't want to expose as a public API - things like internal utilities, database models, or business logic that shouldn't be used outside the project. The key is understanding that `internal/` creates a compile-time boundary. This helps me design clean APIs and prevents accidental coupling between projects. It's Go's way of enforcing encapsulation at the package level."

3.  Replace Implementation.

### Explanation

### How to Explain in Interview (Spoken style format)

**Your Response:** "I refactor legacy Go code systematically. Then I extract interfaces to define clean contracts between components. Finally, I replace implementations piece by piece while maintaining the interface. This approach lets me refactor safely without breaking existing functionality. I might start with the most critical or most problematic areas first. The key is having a safety net of tests before making changes. I also look for common issues like global state, tight coupling, and missing error handling. This methodical approach reduces risk and makes refactoring predictable and manageable."

### Question 1000: How do you conduct performance reviews for Go codebases?

**Answer:**
Check:
- Pointer usage (Escape analysis).
- Lock contention.
- Database N+1.
- Error wrapping.
- Test coverage.

### Explanation
Go codebase performance reviews check pointer usage with escape analysis, identify lock contention issues, detect database N+1 query problems, verify proper error wrapping, and ensure adequate test coverage for performance-critical code.

### How to Explain in Interview (Spoken style format)
**Interviewer:** How do you conduct performance reviews for Go codebases?
**Your Response:** "I conduct performance reviews by checking several key areas. I analyze pointer usage with escape analysis to identify unnecessary heap allocations. I look for lock contention issues that might cause goroutines to block. I check for database N+1 query patterns that can hurt performance. I verify proper error wrapping to avoid performance overhead from repeated error creation. I also review test coverage for performance-critical paths. I use pprof to profile CPU and memory usage, looking for hotspots and allocation patterns. The key is understanding both the Go-specific performance characteristics and the application's specific bottlenecks. I also review recent changes for performance regressions and suggest optimizations based on the findings."
