## ⚒️ Tooling, Maintenance & Real-world Scenarios (Questions 981-1000)

### Question 981: How do you create internal packages in Go?

**Answer:**
Folder named `internal/`.
Go compiler forbids importing `myproject/internal/foo` from `otherproject`.
Enforces boundaries.

---

### Question 982: How do you enforce code standards using `golangci-lint`?

**Answer:**
Create `.golangci.yml`. Enable `revive`, `gocritic`, `errcheck`.
Run in CI. Block PRs on failures.

---

### Question 983: How do you write makefiles for Go projects?

**Answer:**
Standard targets: `build`, `test`, `lint`, `clean`.
```makefile
build:
    go build -o bin/app ./cmd/app
```

---

### Question 984: How do you manage secrets using Vault in Go?

**Answer:**
Use `hashicorp/vault/api`.
Authenticate (K8s Auth).
`client.Logical().Read("secret/data/myapp")`.

---

### Question 985: How do you deploy a Go app with Kubernetes?

**Answer:**
Dockerfile -> Build Image -> Push Registry.
Deployment YAML -> `kubectl apply`.
Service -> Ingress.

---

### Question 986: How do you perform zero-downtime deployment in Go?

**Answer:**
**Rolling Update** (K8s).
Graceful Shutdown (Handle SIGTERM) ensures in-flight requests finish while new pods start up.

---

### Question 987: How do you refactor legacy Go code?

**Answer:**
1.  Add Test Coverage (Characterization Tests).
2.  Extract Interface.
3.  Replace Implementation.

---

### Question 988: How do you organize large-scale Go monorepos?

**Answer:**
**Bazel** or **Buck2** build systems (Google style).
Or standard `go.work` (Go 1.18+).
Shared `pkg/` folder. Distinct `services/` folders.

---

### Question 989: How do you distribute Go binaries securely?

**Answer:**
Sign them (Cosign/GPG).
Serve over HTTPS.
Provide checksums (SHA256).

---

### Question 993: How do you build CLI-based installers in Go?

**Answer:**

### Question 994: How do you generate dashboards from Go metrics?

**Answer:**
Prometheus scrapes Go app.
Grafana queries Prometheus (`rate(http_requests_total[5m])`).

### Question 995: How do you monitor file system changes in Go?

**Answer:**
`fsnotify/fsnotify`.
Uses `inotify` (Linux) or `FSEvents` (Mac).
Watch config file for live-reloading.

### Question 999: How do you migrate Go modules across repos?

**Answer:**
Keep git history? `git subtree`.
Just code? Copy files, grep/replace import paths.

### Question 1000: How do you conduct performance reviews for Go codebases?

**Answer:**
Check:
- Pointer usage (Escape analysis).
- Lock contention.
- Database N+1.
- Error wrapping.
- Test coverage.
