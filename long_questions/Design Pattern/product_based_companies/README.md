# Design Pattern Interview Questions - Product Based Companies

This section contains advanced Design Pattern interview questions and architectural concepts commonly asked in product-based companies (e.g., Amazon, Microsoft, Google, Meta, Uber, Netflix, Stripe).

Product-based companies typically assess a candidate's ability to apply patterns to solve real-world system design challenges, focus heavily on multi-threading/concurrency in patterns, and delve into modern architectural patterns in distributed systems.

## Topics Covered:
1. **[Advanced Creational Patterns](./01_advanced_creational_patterns.md)** (Thread Safety, Framework internals)
2. **[Advanced Structural Patterns](./02_advanced_structural_patterns.md)** (Performance optimization, Middleware)
3. **[Advanced Behavioral Patterns](./03_advanced_behavioral_patterns.md)** (Event-driven architectures, Functional patterns)
4. **[Architectural and System Design Patterns](./04_architectural_patterns.md)** (Microservices, Resiliency patterns)

## Focus Areas for Product-Based Companies
*   **Concurrency & Thread Safety:** How to make the Singleton pattern truly thread-safe and resilient against Reflection and Serialization.
*   **Real-world System Design:** Designing a parking lot, elevator system, or a cache using appropriate patterns.
*   **Framework Knowledge:** How design patterns are used internally in popular frameworks (e.g., Spring Dependency Injection, Proxy/AOP, React's state management).
*   **Performance:** Using patterns like Flyweight for memory optimization at scale.
*   **Modern Paradigms:** Implementing patterns using modern language features (lambdas in Java, closures in JS/Go) and architectural shifts (Microservices patterns like CQRS, Saga).
