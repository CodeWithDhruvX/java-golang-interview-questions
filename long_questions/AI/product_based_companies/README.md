# AI Interview Questions for Product-Based Companies

Welcome to the **Product-Based Companies** section for Artificial Intelligence. Top-tier companies (FAANG, leading startups, and major tech firms globally and in India) require a deep understanding of core algorithms, mathematical foundations, system design, and the latest advancements in GenAI and MLOps.

These interviews go beyond using APIs and frameworks; they test your ability to build from scratch, scale systems, debug deep learning models, and design complex AI architectures.

## Structure

This section is divided into the following key files:

1.  **[Core Machine Learning & Algorithms](./01_machine_learning_core_and_algorithms.md)**: Deep dive into mathematical foundations, core algorithms (XGBoost, SVMs, Trees), loss functions, and optimization techniques.
2.  **[Deep Learning, NLP & Computer Vision](./02_deep_learning_nlp_cv.md)**: Network architectures, CNNs, RNNs/LSTMs, Backpropagation in depth, vanishing gradients, attention mechanisms (pre-transformer), and specialized CV/NLP tasks.
3.  **[Generative AI, Transformers & LLMs](./03_generative_ai_and_llms.md)**: Transformer architecture deep dive, pre-training objectives, Fine-Tuning strategies (LoRA, QLoRA, PEFT), RLHF, DPO, and understanding KV Cache and positional embeddings.
4.  **[AI System Design](./04_ai_system_design.md)**: Designing large-scale AI systems (e.g., Recommendation Systems like Netflix/TikTok, Search Engines, Ad-Click Prediction). Balancing latency, throughput, and hardware optimization (GPUs/TPUs).
5.  **[AI Agents & Advanced RAG](./05_ai_agents_and_rag.md)**: Building autonomous agents, defining tool use, ReAct frameworks, Graph RAG, Agentic reasoning, and navigating vector databases at scale.
6.  **[MLOps, Serving & Deployment](./06_mlops_and_deployment.md)**: Model serving (vLLM, TensorRT-LLM, TorchServe), handling data drift, CI/CD for ML, feature stores, and distributed training (Data Parallelism vs. Tensor/Pipeline Parallelism).
7.  **[PyTorch & ML Coding](./07_pytorch_and_ml_coding_interviews.md)**: Live coding rounds. Writing PyTorch training loops from scratch, implementing Self-Attention dimensions, and building custom loss functions natively.
8.  **[Advanced Statistics & Pure Math](./08_advanced_statistics_and_pure_math.md)**: Proving why MSE = MLE, deriving the derivative of Categorical Cross-Entropy + Softmax, and understanding the KL Divergence math inside RLHF.

## Interview Expectations

For high-paying roles (e.g., Machine Learning Engineer, AI Researcher, Applied Scientist, AI Architect):
-   **First Principles Thinking**: You must explain *why* an algorithm works, not just *how* to use it in PyTorch/Scikit-Learn.
-   **Math**: Be prepared to write down loss functions, derive gradients, and explain probability distributions.
-   **Scale**: Expect questions on how your model will behave serving millions of requests per second.
-   **Trade-offs**: You will constantly be asked to discuss trade-offs (e.g., Latency vs. Accuracy, RAM vs. Compute, RAG vs. Fine-Tuning).
