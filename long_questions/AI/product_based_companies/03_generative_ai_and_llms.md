## 1. Explain the Self-Attention mechanism mathematically. What are Query, Key, and Value matrices?
**Answer:**
Self-attention allows a model to weigh the importance of other words in an input sequence when encoding a specific word.

**The Q, K, and V concept (borrowed from database retrieval):**
Imagine you want to search for a video on YouTube. You type a **Query** ("funny cats"). The search engine compares your Query to the **Keys** (video titles/tags) of all available videos. Based on the similarity, it returns the most relevant **Values** (the video contents).

In an LLM, for every token in our sequence, we generate three vectors:
1.  **Query ($Q$):** What I am looking for.
2.  **Key ($K$):** What I contain.
3.  **Value ($V$):** My actual semantic meaning/content.
These are generated by multiplying the input embedding $X$ by learned weight matrices: $W_Q, W_K, W_V$.

**The Math (Scaled Dot-Product Attention):**
$$ \text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V $$

Let's break it down:
1.  **$QK^T$ (Dot Product):** We calculate the dot product between the Query of the current word and the Keys of *all* other words in the sequence. A higher dot product means higher similarity/relevance.
2.  **$\frac{1}{\sqrt{d_k}}$ (Scaling):** We divide by the square root of the dimension of the key vectors. If $d_k$ is large, dot products can grow huge, pushing the softmax function into regions where gradients are extremely small (vanishing gradient). Scaling prevents this.
3.  **$\text{softmax}(\dots)$:** Converts the raw scores into probabilities that sum to 1. These are the *attention weights*.
4.  **$\dots \times V$:** We multiply the attention weights by the Value vectors and sum them up. The resulting vector is a rich representation of the current word, incorporating context from the entire sequence.

## 2. Why do Transformers require Positional Embeddings? How do they work?
**Answer:**
**The Problem:**
Unlike RNNs which process tokens sequentially step-by-step, or CNNs which process patches spatially, the Self-Attention mechanism in a Transformer processes the entire sequence in parallel. The operation $QK^T$ has no inherent concept of order. Without positional information, the model would process the sentence "The dog bit the man" exactly the same as "The man bit the dog" (treating it merely as a "bag of words").

**The Solution:**
Positional Embeddings explicitly inject sequence order information into the model.

**How they work:**
Before pushing the word embeddings into the Transformer, we add a positional vector to them:
$\text{Final\_Embedding} = \text{Word\_Embedding} + \text{Positional\_Embedding}$

*   **Original (Sinusoidal) Positional Encodings:** The original "Attention is All You Need" paper used deterministic sine and cosine functions of varying frequencies based on the position $pos$ and the dimension index $i$.
    *   $PE_{(pos, 2i)} = \sin(pos / 10000^{2i/d_{model}})$
    *   $PE_{(pos, 2i+1)} = \cos(pos / 10000^{2i/d_{model}})$
    *   *Advantage:* The model can easily learn to attend by relative positions because a linear function can represent the offset between two positions. Furthermore, it can cleanly extrapolate to sequence lengths longer than seen in training.
*   **Learned Positional Embeddings:** Many modern LLMs (like standard BERT/GPT-2) simply treat position indices as an extra vocab layer and learn an embedding for "Position 1," "Position 2," etc., during training.
*   **RoPE (Rotary Position Embeddings):** Used in LLaMA. Instead of adding a vector, RoPE *rotates* the Q and K vectors in a mathematically specific way based on their positions before calculating the dot product. It beautifully captures relative distance natively within the attention calculation.

## 3. Walk me through the typical LLM training pipeline: Pre-training, SFT, and RLHF/DPO.
**Answer:**
The creation of a modern aligned LLM (like ChatGPT or Claude) involves three distinct phases:

1.  **Pre-training (The "Reading the Internet" Phase):**
    *   **Goal:** Learn the statistical structure of language, grammar, facts, and reasoning capabilities.
    *   **Data:** Massive unstructured datasets (Terabytes/Petabytes) mapping the web, Wikipedia, books, GitHub (e.g., Common Crawl).
    *   **Objective:** Next-Token Prediction (Causal Language Modeling). Given context $X$, predict token $y$.
    *   **Output:** A "Base Model" (e.g., Llama-3-Base). It knows a lot but isn't helpful. If asked "How to bake a cake?", it might respond with "How to bake a cookie?" because it's just continuing the pattern of questions.

2.  **Supervised Fine-Tuning (SFT) (The "Format Teaching" Phase):**
    *   **Goal:** Teach the model to act as a conversational assistant and follow instructions.
    *   **Data:** High-quality, human-curated prompt-response pairs (tens to hundreds of thousands of examples).
    *   **Objective:** Next-Token Prediction, but specifically trained on the *response* part conditioned on the *prompt*.
    *   **Output:** An "Instruct Model". It now answers questions properly but might hallucinate, be overly verbose, or give unsafe advice.

3.  **Alignment: RLHF or DPO (The "Preference Teaching" Phase):**
    *   **Goal:** Align the model's outputs with human values (helpfulness, honesty, harmlessness) and preferences.
    *   **RLHF (Reinforcement Learning from Human Feedback):**
        1.  *Reward Model Training:* Generate multiple responses to a prompt using the SFT model. Humans rank them (A is better than B). Train a separate "Reward Model" to predict these human scores.
        2.  *PPO (Proximal Policy Optimization):* Use the SFT model to generate a response. Pass it to the Reward Model to get a score. Use Reinforcement Learning (PPO algorithm) to update the LLM's weights to maximize this reward, while ensuring it doesn't drift too far from the original SFT model (using KL divergence).
    *   **DPO (Direct Preference Optimization):** A newer, more stable alternative to RLHF. Instead of training a separate reward model and doing complex RL, DPO uses a mathematical trick to optimize the LLM *directly* on the human preference data using a simple modified cross-entropy loss.

## 4. What is LoRA (Low-Rank Adaptation) and mathematically why does it save so much VRAM during fine-tuning?
**Answer:**
**The Problem:**
Full fine-tuning of an LLM (e.g., a 7B parameter model) requires updating all the original weights $W_0$. This means we need to store the gradients and optimizer states (like Adam moments) for all 7 Billion parameters, which requires massive VRAM (often $>4\times$ the model size).

**The LoRA Solution:**
LoRA freezes the pre-trained model weights ($W_0$) and instead injects trainable "rank decomposition matrices" into the layers of the Transformer architecture.

**The Math:**
During fine-tuning, an updated weight matrix can be represented as $W = W_0 + \Delta W$.
LoRA hypothesizes that the weight updates ($\Delta W$) happen in a "low intrinsic dimension" space. Instead of learning the full matrix $\Delta W$ (which is $d \times k$), LoRA approximates it by multiplying two much smaller matrices, $B$ and $A$:
$$ \Delta W \approx B \times A $$
Where:
*   $B$ is of size $d \times r$
*   $A$ is of size $r \times k$
*   $r$ (the rank) is a small number (e.g., 8, 16, or 64).

**Why it saves VRAM:**
Imagine a weight matrix $W_0$ in a self-attention layer of size $4096 \times 4096$ (over 16 million parameters).
If we use LoRA with rank $r=8$:
*   Matrix $B$ is size $4096 \times 8$ ($\approx 32,000$ params)
*   Matrix $A$ is size $8 \times 4096$ ($\approx 32,000$ params)
Total trainable parameters = $64,000$.
We reduced the number of trainable parameters by a factor of 250! We only need to store gradients and optimizer states for these $64,000$ parameters, drastically reducing VRAM requirements to the point where models can be fine-tuned on consumer GPUs.

During inference, we simply add $B \times A$ back into $W_0$ ($W_{final} = W_0 + B \times A$), resulting in zero added latency.

## 5. What is the KV Cache during LLM inference? Why is it a bottleneck, and how do techniques like PagedAttention help?
**Answer:**
**What is KV Cache:**
LLM generation is autoregressive: we generate token $t_1$, then input $t_1$ to generate $t_2$, and so on.
During Self-Attention, to generate the next token, we compute the Query ($Q$) for the current token and compare it to the Keys ($K$) and Values ($V$) of *all previous tokens* in the context.
Recomputing the $K$ and $V$ matrices for all historical tokens at every single step is immensely wasteful.
The **KV Cache** solves this by physically storing the $K$ and $V$ tensors for past tokens in VRAM. Next time we generate a token, we only compute $Q, K, V$ for the *new* token, append its $K$ and $V$ to the cache, and do attention over the cached history.

**Why it's a bottleneck:**
While it speeds up compute (solving the compute bound), it shifts the problem to being **Memory Bound**. The KV cache size grows linearly with: `Batch Size × Context Length × Layers × Hidden Dimension`. For large models with large batch sizes, the KV cache can consume more VRAM than the model weights themselves, strictly limiting how many concurrent users a server can handle.

**The PagedAttention Solution (vLLM):**
Traditional KV caching pre-allocates contiguous chunks of VRAM for the maximum possible sequence length. This leads to massive **internal fragmentation** (wasted VRAM if generating a short response) and **external fragmentation** (inability to fit new requests even if total free VRAM exists).

PagedAttention borrows from OS virtual memory paging. It divides the KV cache into small, fixed-size blocks (e.g., holding 16 tokens).
*   Tokens are parsed dynamically into these blocks.
*   The blocks don't need to be physically contiguous in VRAM.
*   A "Block Table" maps logical token blocks to physical VRAM locations.
*   This largely eliminates memory fragmentation, allowing the server to fit significantly larger batch sizes and increasing throughput by $2\times$ to $4\times$.
