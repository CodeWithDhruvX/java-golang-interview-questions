# AI Interview Questions for Service-Based Companies

Welcome to the **Service-Based Companies** section for Artificial Intelligence. Interviews for service-based companies (e.g., TCS, Infosys, Wipro, Cognizant, Accenture, and specialized AI consultancies) often prioritize practical, applied knowledge over deep mathematical theory.

The expectation is that you can quickly understand a client's requirements, choose the right off-the-shelf tools (Scikit-Learn, Hugging Face, OpenAI APIs, Cloud platforms), and deliver a working prototype or production-ready application.

## Structure

This section is divided into the following key files:

1.  **[AI Basics & Concepts](./01_ai_basics_and_concepts.md)**: Fundamental terminology (AI vs. ML vs. DL), types of learning (Supervised/Unsupervised), evaluation metrics (Accuracy, Precision, Recall, F1), and basic model trade-offs.
2.  **[Applied Machine Learning](./02_applied_machine_learning.md)**: Practical utilization of Python libraries (Pandas, Scikit-Learn). Focuses on data preprocessing, missing value imputation, standard models (Random Forest, Logistic Regression), and Cross-Validation.
3.  **[Prompt Engineering & API Usage](./03_prompt_engineering_and_api.md)**: Knowing your way around the OpenAI/Anthropic APIs, building robust prompts, Few-Shot vs. Zero-Shot prompting, handling JSON outputs, and managing token constraints.
4.  **[Cloud AI Services](./04_cloud_ai_services.md)**: Practical knowledge of utilizing pre-built AI services in the cloud (AWS SageMaker/Rekognition, Azure OpenAI/Cognitive Services, GCP Vertex AI).
5.  **[Basic RAG & Vector Databases](./05_basic_rag_and_vector_databases.md)**: How to build a simple Retrieval-Augmented Generation system using LangChain/LlamaIndex, setting up a naive Vector Database (ChromaDB, Pinecone), and answering questions from a PDF.

## Interview Expectations

For roles like Data Scientist, AI Consultant, ML Developer:
-   **Implementation:** You will likely be asked to write Python code (or pseudo-code) using standard libraries like `pandas` or `sklearn`.
-   **Client Scenarios:** "A retail client wants to predict customer churn. What data do you ask for, and what model do you use?"
-   **Integration:** Focus heavily on integrating APIs into existing applications (e.g., adding a chatbot feature to a Spring Boot or Node.js backend).
-   **Broad Knowledge:** Knowing a little bit about many tools (AWS, Azure, LangChain, GenAI) is often preferred over extreme depth in just one mathematical algorithm.
