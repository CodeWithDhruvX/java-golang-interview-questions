# 🏢 Java Interview Questions — Service-Based Companies

> Companies like **TCS, Infosys, Wipro, Cognizant, Capgemini, Accenture, HCL, Tech Mahindra**, etc.

Service-based companies focus on **practical Java skills**, building REST APIs, working with Spring Boot, integrating with databases and messaging systems. Interviews are **moderate in depth** and emphasize real-world engineering over deep internals.

## 🎯 Interview Focus Areas (Service Companies)

- ✅ Java fundamentals: OOP, generics, streams, lambdas, exception handling
- ✅ Spring Boot: dependency injection, auto-configuration, bean lifecycle
- ✅ REST APIs: CRUD endpoints, validation, error handling, pagination
- ✅ JPA/Hibernate: entity mappings, N+1 problem, lazy/eager loading, transactions
- ✅ Kafka basics: topics, producers, consumers, consumer groups
- ✅ Microservices: service discovery, API gateway, circuit breaker
- ✅ Concurrency: threads, synchronized, ExecutorService, CompletableFuture

---

## 💡 Tips for Service Company Java Interviews

1. **Know Spring Boot deeply** — DI, `@Transactional`, bean scopes, profiles
2. **Master JPA** — entity relationships, fetch types, N+1 fix with JOIN FETCH
3. **Practice REST API design** — proper HTTP methods, status codes, DTO pattern
4. **Understand Kafka at a conceptual level** — partitions, consumer groups, offsets
5. **Explain microservices trade-offs** — when to use vs monolith, CAP theorem
6. **Know OOP well** — abstract vs interface, overloading vs overriding, SOLID
7. **Thread safety basics** — `synchronized`, `volatile`, `AtomicInteger`, `ConcurrentHashMap`
