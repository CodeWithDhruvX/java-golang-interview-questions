# Object-Oriented Design (LLD) - Product Based Companies

This section contains common Low-Level Design (LLD) or Object-Oriented Design (OOD) interview questions asked by top product-based companies (e.g., FAANG, Uber, Airbnb).

These questions generally test your ability to model a system using object-oriented principles (Encapsulation, Polymorphism, Abstraction, Inheritance), apply appropriate Design Patterns (Strategy, Observer, Factory, Singleton, Command), and write thread-safe, modular, and maintainable code.

## Success Criteria for LLD Interviews
*   **Clarify Ambiguity:** Gather use cases and constraints.
*   **Object Modeling:** Define core entities, attributes, and relationships (composition, association).
*   **API/Methods:** Define class interfaces and behavior.
*   **Design Patterns:** Apply patterns appropriately (do not over-engineer).
*   **Concurrency:** Address thread-safety when handling shared resources (locks, synchronized blocks, Concurrent collections).
