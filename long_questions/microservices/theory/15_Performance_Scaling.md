### 244. What is non-blocking IO?
"Blocking I/O implies that a thread makes a network request and completely halts execution (sleeping) until the response returns. This wastes the thread entirely.

**Non-Blocking I/O** (NIO), utilized heavily in Node.js or Java Netty, fundamentally changes this. 
When a thread makes a database request, it does not wait. It registers a 'Callback' and immediately executes other tasks. When the database finally responds, an 'Event Loop' interrupts the thread, saying, 'Your data is ready, here is the payload.'

This allows a single OS thread to seamlessly manage 10,000 active concurrent connections effortlessly without crashing."

#### Indepth
Spring WebFlux was built exactly on this paradigm. While it drastically enhances concurrent throughput, it historically turns code into "Callback Hell". Java's recent Virtual Threads completely solve this by allowing developers to write normal, synchronous-looking code while the JVM seamlessly physically handles the Non-Blocking I/O transparently in the background, achieving the best of both worlds natively.
